## 2024-03-24 - Fix JSON API endpoints using incorrect authentication middleware
**Vulnerability:** Several JSON API endpoints (`/api/shop/exchange`, `/api/shop/buy`, `/api/inventory/equip`, `/api/inventory/sell`, `/api/ah/buy`, `/api/ah/list`) were protected by the `s.auth` middleware instead of the appropriate `s.authAPI` middleware. This causes unauthenticated requests to receive an HTML redirect (HTTP 303) rather than a proper JSON error response.
**Learning:** The web server has dual authentication middleware: `s.auth` for HTML pages (redirects to `/denied`) and `s.authAPI` for JSON APIs (returns `{"ok": false, "error": "unauthenticated"}`). Mixing these up can lead to client-side errors and unpredictable behavior when an API client expects JSON but receives an HTML redirect.
**Prevention:** Ensure that any route serving a JSON API uses the `s.authAPI` middleware, while routes serving HTML pages use the `s.auth` middleware. Always review the expected return type of an endpoint when applying an authentication wrapper.
