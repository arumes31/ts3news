## 2024-05-19 - Keyboard Accessibility Focus Indicators
**Learning:** Found that custom-styled `button`s, `a` tags, `input` fields, and `.lb-cat-btn` elements lack sufficient visual feedback when navigated via keyboard. Without explicit focus rings, users utilizing keyboard navigation are unable to track their current position within the portal, causing severe accessibility and usability issues. This is especially true for custom elements where default browser focus styles are overridden or non-obvious against the site's dark theme.
**Action:** Always provide explicit `:focus-visible` styles using `outline` and `outline-offset` to all interactive UI components, matching the application's primary accent colour (e.g. `var(--accent)`) to ensure prominent visual feedback exclusively for keyboard users.
